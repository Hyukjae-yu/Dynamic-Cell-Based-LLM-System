================================================================================
오프라인 민지 시스템 (vShell & vCPU) 통합 사용자 매뉴얼
================================================================================

본 매뉴얼은 가상 메모리(mmap) 기반의 의미셀(Semantic Cell) 추론망과 vCPU 스크립트 
실행 엔진, 그리고 실시간 시스템 시계/타이머가 결합된 통합 시스템의 동작 원리 및 
전체 명령어 명세서입니다.

--------------------------------------------------------------------------------
1. 시스템 아키텍처 개요
--------------------------------------------------------------------------------

본 시스템은 영구적인 가상 메모리 파일(system.bin)을 mmap으로 메모리에 매핑하여 
커널 레벨의 빠른 속도와 전역 공유 상태를 유지합니다.

* 의미셀 추론망 (Semantic Network)
  - 노드(Cell)와 연결 가중치(Synapse)로 구성된 신경망형 데이터베이스입니다.
  - 각 셀은 에너지(Energy)와 임계값(Pth, Nth)을 가집니다.
  - 에너지가 전파되면서 조건 충족 시 등록된 vCPU 스크립트를 비동기적으로 
    실행(인터럽트)합니다.

* 가상 변수 관리자 (vMalloc)
  - 128KB의 비트맵(varBitmap)을 이용해 1MB 크기의 데이터 공간(varData)을 
    100% 동적, 자동 관리합니다.
  - 사용자가 변수를 정의하면 고유한 32비트 가상 주소로 인코딩되어 전역 
    매핑 테이블(globalVarTable)에 영구 고정 관리됩니다.

* vCPU 실행 엔진 (Virtual Execution Engine)
  - 의미셀에 가상 레지스터 기반 스크립트를 바인딩합니다.
  - 셀이 발화할 때 인터럽트 방식으로 스크립트를 호출하는 경량 가상 프로세서입니다.

--------------------------------------------------------------------------------
2. vShell 터미널 모드 명령어 명세
--------------------------------------------------------------------------------

vShell 콘솔에 직접 입력하여 실행하는 시스템 제어 명령어 목록입니다. 
모든 명령어는 대소문자를 구분합니다.

[2.1. 의미셀(Cell) 상태 및 속성 제어]
* create [셀이름]
  - 새로운 의미셀을 생성합니다. (예: create 알람수신셀)
* delete [셀이름]
  - 지정된 의미셀을 영구 삭제합니다.
* [셀이름].energy [연산자] [값]
  - 셀의 현재 에너지를 직접 설정하거나 가감합니다. 
  - 연산자는 =, +, - 가 가능합니다. (예: 철수.energy = 1.0)
* [셀이름].Pth [값]
  - 정방향 발화 임계값(Positive Threshold)을 설정합니다. 
  - 에너지가 이 값 이상이 되면 셀이 활성화 및 발화합니다.
* [셀이름].Nth [값]
  - 부방향 임계값(Negative Threshold)을 설정합니다.
* [셀이름].exec [true/false]
  - 셀에 바인딩된 vCPU 스크립트의 실행 허용 여부를 설정합니다.
* [셀이름].active [true/false]
  - 셀의 활성화 상태를 강제로 설정합니다.

[2.2. 시냅스(Synapse) 연결 및 제어]
* link_fact [출발셀] [도착셀] [가중치]
  - 두 셀 사이에 지정된 가중치를 가진 시냅스 링크를 생성합니다. 
  - (예: link_fact 배고픔 식사 0.8)
* unlink [출발셀] [도착셀]
  - 두 셀 사이의 시냅스 연결을 끊습니다.
* decay_links [셀이름] [감쇠율]
  - 지정된 셀과 연결된 모든 링크의 가중치를 일정 비율로 감소(망각)시킵니다.
* [출발셀].([도착셀]) [연산자] [가중치]
  - 두 셀 사이의 가중치를 직접 조절합니다. (예: 배고픔.(식사) = 0.95)

[2.3. 추론망 제어 및 모니터링]
* propagate [단계수]
  - 현재 활성화된 셀들의 에너지를 링크 가중치에 따라 지정된 단계만큼 전방위로 
    전파(Propagate)시킵니다.
* reset_energy_all
  - 네트워크 상의 모든 셀들의 에너지를 0.0으로 일괄 초기화합니다.
* fire [셀이름]
  - 지정된 셀에 에너지를 주입하여 강제로 발화(인터럽트 실행)를 트리거합니다.
* print_fired
  - 현재 발화 조건(에너지 >= Pth)을 만족한 셀들의 목록을 콘솔에 출력합니다.

[2.4. 가상 전역 변수 선언 및 관리]
* var [변수명]
  - 전역 가상 변수 테이블에 새로운 변수를 동적으로 선언 및 등록합니다.
  - 접두사 규칙: n으로 시작하면 정수형, f로 시작하면 고정소수점형, U1_로 시작하면 
    시스템 예약형 변수로 등록됩니다.
* print "[포맷문자열]" [변수1] [변수2]
  - 변수의 현재 값을 지정된 포맷으로 화면에 출력합니다.
  - 따옴표("")를 사용하여 공백이 포함된 문장을 안전하게 출력할 수 있습니다.
  - (예: print "철수 잔액: %f 원" n철수잔액)

[2.5. 파일 I/O 및 파일 제어 유틸리티]
* file_load [파일명] [크기] [대상오프셋/변수]
  - 디스크의 파일을 가상 메모리(varData)의 특정 물리 오프셋 또는 지정 변수 
    위치로 읽어옵니다.
* file_save [파일명] [크기] [소스오프셋/변수]
  - 가상 메모리의 특정 영역 데이터를 파일로 저장합니다.
* file_append [파일명] [크기] [소스오프셋/변수]
  - 가상 메모리의 특정 영역 데이터를 기존 파일의 끝에 이어붙입니다.
* file_delete [파일명]
  - 가상의 파일 시스템 및 디스크에서 해당 파일을 영구 삭제합니다.

[2.6. 기타 시스템 제어]
* sync
  - 현재 가상 메모리 공간(mmap)의 데이터 상태를 디스크 마스터 파일(system.bin)에 
    강제로 동기화(msync)합니다.
* echo [메시지]
  - 콘솔에 지정한 텍스트를 그대로 메아리 출력합니다.
* wait [ms]
  - vShell 동작을 지정된 밀리초(ms) 단위 시간만큼 일시 중지합니다.
* manual
  - 간이 도움말을 화면에 출력합니다.

3. 코맨드라인 실시간 사칙연산 및 자동 등록
--------------------------------------------------------------------------------

vShell 콘솔 창에서는 별도의 var 선언 명령 없이도 즉석에서 새로운 전역 변수를 
대입식 하나로 자동 선언하고 동시에 사칙연산 결과를 할당할 수 있습니다.

[3.1. 대입을 통한 자동 변수 등록]
* 변수 테이블에 등록되지 않은 임의의 변수명을 = 기호와 함께 대입식으로 입력하면, 
  시스템이 알아서 빈 공간을 찾아 메모리를 할당(vMalloc)하고 변수 매핑 테이블에 
  등록한 뒤 대입 연산을 수행합니다.
* 띄어쓰기 유무 상관없음: 
  'n철수월급=1000000'과 'n철수월급 = 1000000'은 완벽히 동일하게 작동합니다.

[3.2. 실시간 인라인 사칙연산]
변수 할당 연산 시 기본 사칙연산 기호(+, -, *, /)와 실수를 조합하여 즉석 계산된 
값을 할당할 수 있습니다. 피연산자 자리에는 상수(정수/실수)뿐만 아니라 이미 선언된 
전역 변수명도 자유롭게 사용 가능합니다.

* n철수월급 = 1000000 (자동 등록 및 초기 할당)
* n철수보너스 = n철수월급 * 0.5 (기존 전역 변수 연동 연산)
* f이자율 = 0.05
* f수익금 = 50000 * f이자율 (고정소수점 변수 연계 연산)
* n철수일급 = n철수월급 / 30 (나눗셈 연산)

[3.3. 단일 변수 값 조회]
* 대입 연산자 없이 전역 변수명만 콘솔에 입력하면 현재 메모리에 저장된 원시값 및 
  실제 디코딩 값을 즉석에서 읽어와 콘솔에 출력합니다. 
  (예: 'n철수월급' 입력 시 '-> [vShell] 'n철수월급' 현재 값: 1000000' 출력)

--------------------------------------------------------------------------------
4. 실시간 시스템 시계 & 8채널 타이머 인터럽트
--------------------------------------------------------------------------------

배경(Background Monitor) 스레드에서 10ms 단위로 실시간 폴링하며 시간을 계측하고, 
지정된 타이머 만료 시 vCPU 인터럽트를 트리거하는 핵심 장치입니다.

[4.1. 가상 메모리 레이아웃 고정 예약]
시스템 부팅 시 varData 가상 메모리 공간의 최상단 72바이트는 아래 변수들의 
용도로 강제 예약 및 글로벌 테이블에 고정 매핑됩니다.

* 0번 ~ 7번 바이트 (8B): U1_sysClock (시스템 시간 저장)
* 8번 ~ 71번 바이트 (64B): U1_sysTimer0 ~ U1_sysTimer7 (8개 채널 타이머)

[4.2. 시스템 시계 (U1_sysClock)]
* 시스템 전체에서 공유되는 실시간 100ms 단위의 단조 증가 타임스탬프입니다.
* 사용자는 읽기 전용으로 접근하여 현재 시스템 동작 시간 흐름을 제어할 수 있습니다.

[4.3. 8채널 가상 타이머 (U1_sysTimer0 ~ U1_sysTimer7)]
* 디폴트 상태 (OFF): -1 (18446744073709551615) 값으로 채워져 비활성화되어 있습니다.
* 작동 구조 (64비트 공간 압축 레이아웃):
  [ 상위 32비트 : 타겟 셀 ID (CellID) ] | [ 하위 32비트 : 예약 딜레이 시간 (100ms 단위) ]

* 타이머 세팅 및 작동 순서:
  1. 사용자가 타이머 변수에 압축된 64비트 데이터를 기록합니다. 
     (예: U1_sysTimer0 = 21474836490)
  2. 배경 스레드가 이를 감지하여 내부 타겟 만료 시간(현재 시간 + 딜레이)을 수립합니다.
  3. 그 후, 해당 타이머 값을 주기적으로 읽어올 때마다 실시간 남은 카운트다운 숫자가 반환됩니다.
  4. 카운트다운이 0에 도달하면, 지정된 타겟 셀(CellID)의 에너지를 1.0으로 인젝션하여 
     바인딩된 vCPU 스크립트를 즉시 실행하고, 해당 타이머를 자동으로 디폴트 OFF 상태(-1)로 되돌립니다.

5. 스크립트 에디터 모드 (edit)
--------------------------------------------------------------------------------

vShell 내부에는 간이 파일 및 스크립트를 터미널에서 즉석으로 코딩 및 편입할 수 있는 
라인 에디터 기능이 내장되어 있습니다.

[5.1. 에디터 진입 및 편집 명령어]
* edit [파일명]
  - 스크립트 에디터 환경으로 전환합니다. 지정한 파일명이 존재하지 않으면 신규 생성하고, 
    존재하면 기존 내용을 라인 단위로 로드합니다.
  - 에디터 모드로 진입하면 일반 쉘 명령어는 일시 중단되며, 터미널 프롬프트가 대기 상태로 
    전환되어 한 줄씩 코드를 입력받아 편집 버퍼에 누적합니다.

[5.2. 에디터 모드 전용 메타 명령]
에디터 화면에서 코드 라인 대신 아래 예약 명령을 입력하면 특수 동작을 수행합니다.
* save
  - 현재까지 버퍼에 편집 중인 소스 코드를 지정한 파일명(.vsh)으로 디스크에 저장합니다.
* exit
  - 수정 버퍼를 닫고 에디터 모드를 탈출하여 다시 일반 vShell 터미널 모드로 돌아옵니다.
* compile
  - 현재 작성된 스크립트를 컴파일러를 통해 바이너리 명령어 패킷 파일(.vbin)로 
    컴파일 빌드합니다.

--------------------------------------------------------------------------------
6. vCPU 실행 엔진 명령어 명세 (Assembly)
--------------------------------------------------------------------------------

의미셀이 발화하거나, vShell 및 타이머에 의해 실행 명령을 받을 때 실행되는 vCPU 
경량 아키텍처용 인스트럭션 세트입니다.

* mov [대상] [값/소스]
  - 대상 레지스터나 변수에 특정 값 또는 소스 레지스터의 데이터를 복사합니다.
* add [대상] [값/소스]
  - 대상 데이터에 지정한 값 또는 소스 데이터를 더합니다.
* sub [대상] [값/소스]
  - 대상 데이터에서 지정한 값 또는 소스 데이터를 뺍니다.
* mul [대상] [값/소스]
  - 대상 데이터에 곱 연산을 수행합니다.
* div [대상] [값/소스]
  - 대상 데이터를 나눕니다. (0 나누기 방지 예외 처리 내장)
* jmp [레이블]
  - 지정된 레이블(코드 오프셋 위치)로 무조건 분기합니다.
* je [레이블]
  - 이전 연산 결과가 0(Equal)이면 해당 레이블로 조건 분기합니다.
* jne [레이블]
  - 이전 연산 결과가 0이 아니면(Not Equal) 해당 레이블로 조건 분기합니다.
* bind_script [셀이름] [스크립트명]
  - 특정 의미셀에 컴파일된 스크립트 파일을 연결하여 발화 시 자동 호출되도록 세팅합니다.
* trigger_vcpu
  - 등록된 활성 인터럽트 큐의 모든 서브루틴 실행을 즉각 강제 수행합니다.
* ret
  - vCPU 서브루틴 실행을 안전하게 끝마치고 프로그램 제어권을 메인 시스템으로 반환합니다. 
    모든 스크립트의 끝에는 이 명령이 필수적으로 포함되어야 합니다.

7. 시스템 통합 활용 실전 샘플 코드
--------------------------------------------------------------------------------

[7.1. 시나리오 1: 8채널 타이머를 이용한 비동기 알람 인터럽트 구현]

Step 1. vShell 터미널에서 타겟 의미셀을 생성하고 스크립트 에디터를 엽니다.
-----------------------------------------------------------------
create 알람수신셀
edit alarm_script
-----------------------------------------------------------------

Step 2. 에디터 내부에서 타이머 만료 시 실행될 코드를 작성합니다.
-----------------------------------------------------------------
print "[vCPU 알람!] 타이머가 만료되어 '알람수신셀'의 인터럽트 스크립트가 실행되었습니다!"

[7.2. 시나리오 2: 동적 급여 회계 관리 및 고정소수점 사칙연산 추론식]

사전 선언 없이 대입식 하나로 변수를 자동 생성하여 사용하는 실전 예시입니다.
-----------------------------------------------------------------
# 1. 사원 철수의 급여 테이블 동적 수립 및 세금 차감 연산
n철수기본급 = 3500000
f소득세율 = 0.033

# 인라인 사칙연산을 통한 실급여 계산
f세금공제액 = n철수기본급 * f소득세율
n철수실수령액 = n철수기본급 - f세금공제액

# 2. 결과 출력 확인
print "철수의 기본급: %lld 원, 소득세율: %f" n철수기본급 f소득세율
print "원천세 공제금액: %f 원" f세금공제액
print "최종 통장 실수령액: %lld 원" n철수실수령액

# 3. 추가 연계 (적금 적립금 산출)
f적금비율 = 0.40
n철수적금액 = n철수실수령액 * f적금비율
print "매달 적금으로 빠져나갈 돈: %lld 원" n철수적금액
